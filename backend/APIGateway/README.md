# APIGateway

Это микросервис на Java (Spring Cloud Gateway), который является единой точкой входа в мир backend'а, обеспечивает маршрутизацию запросов.

### Аутентификация
Реализована посредством проверки Bearer Token'а.
- Если токен существует как ключ в Redis, то запрос проскакивает дальше, заголовок Authorization удаляется, и ему на замену приходят заголовки "X-User-Id" и "X-Username" для дальнейшей персонализации в микросервисах.
- Если не существует, то пользователь получает ошибку сразу, запрос не проходит дальше.

---
### Rate limiting
Реализован по той же логике: APIGateway ведет счетчик запросов в Redis, и, если запросов за последнюю минуту оказалось больше допустимого, то пользователь получает статус "429 Too Many Requests".

---
### Асинхронность
Здесь используется реактивная WebFlux версия Spring Cloud Gateway, которая предоставляет способы асинхронной обработки запросов. Потоки не блокируются в ожидании ответа от Redis или целевых микросервисов. Вместо этого они сразу переключаются на обработку других запросов, а когда ответ будет готов - выполнение продолжается на любом свободном потоке. Это позволяет одному потоку обслуживать тысячи запросов, в отличие от классического блокирующего подхода Tomcat (один поток на запрос).
