# Документация микросервисов

Здесь находятся Postman-коллекции для тестирования API каждого микросервиса.

## Как использовать

1. Скачайте и откройте desktop версию Postman (и если нужно зарегистрируйтесь).
2. Скачайте нужную вам коллекцию (а лучше сразу все).
3. Зайдите в любой workspace.
4. Перетащите коллекцию из папки в левую пустую область, коллекция импортируется автоматически.

Для чтения документации нажмите на папку коллекции -> в открывшемся окне на "View complete documentation".

Для отправки тестового запроса убедитесь, что в docker compose все нужные сервисы запущены  (см. раздел "Подъем docker compose" в README в корне проекта), а затем откройте нужный запрос, в строке url запроса вставьте в начало `http://localhost:8080` (порт нужно уточнять в соответствии с вашим .env файликом) и нажмите `Send`. Или вместо ручной вставки в начало url'а, можно воспользоваться переменными окружения Postman.

---
## Порядок тестирования

1. Сначала импортируйте `api-gateway.postman_collection.json`
2. Выполните логин через `POST /api/auth/register`
3. Скопируйте полученный токен.
4. Для любых запросов с аутентификацией вручную добавьте скопированный токен во вкладке Authorization -> Token.
